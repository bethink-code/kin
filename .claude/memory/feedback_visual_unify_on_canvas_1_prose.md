When the time comes to unify the rendering of prose-style artefacts across canvases (Canvas 1 picture, Canvas 2 analysis, future Canvas 3 plan, Canvas 4 progress), Canvas 1's `StoryArticle` is the visual reference. Canvas 2's `AnalysisProse` reads worse and should be brought up to match — not the reverse.

**Why:** Garth tested both with users and explicitly preferred Canvas 1's typography + spacing + section rhythm. Canvas 2's prose feels denser / less considered. Editorial, typographic, restrained is the project's visual language (see CLAUDE.md product principles).

**How to apply:**
- Functional parity (annotations, explain pane, chat-driven regenerate, manual refresh) is symmetric — both canvases get the same capabilities.
- Visual rendering: bring Canvas 2 UP to Canvas 1's pattern. Don't downgrade Canvas 1.
- Future canvases (Plan, Progress) when their prose ships, follow Canvas 1's pattern from day one.
- Specific Canvas 1 patterns to preserve: serif H2 for section heads, generous vertical rhythm (`space-y-12`), one-decision-per-paragraph, money/percent rendered with shared formatters, no dense bulleted lists where prose serves better.
